Парметры консоли:

-h, -helpВывод внутренней справки
-count <n>Размер теста. Для первой порядка миллиона, для второй около 64 так как задает порядок квадратной матрицы
-steps <n>Количетво шагов хватить примерно 30
-average <n>Количество циклов усреднения (чем больше тем более гладким должен быть график)
-threads <n>Колчество потоков (по умолчанию выставляется по количеству потоков выполнения)
-lnЭкспоненциалный шаг для отобрадения на логарифмиеской шкале
-silentОтключение вывода только процент выполнения
-graphПостроение графика

Примеры тестов:

График показывающий эффективность мультипоточности

App_1.exe -count 1048576 -steps 32 -average 32 -ln -silent -graph
App_2.exe -count 64 -steps 32 -average 32 -ln -silent -graph
Посмотреть как количество потоков влияет на проиводительность
n можно взять 4/6/8/10/16/32

App_1.exe -count 1048576 -threads <n>
App_2.exe -count 64 -threads <n>
